Project Purpose and Goal

Our client, the Organisation of Africa United, has embarked on a national project to support Early Child Development centres funded by the Department of Social Development. This Project aims to improve access to ECDs across the country while providing quality and efficient services to society. Our client is in need of a system that will not only provide access but will also allow all parties that are needed to make Early Child Development a success to get involved, interact and participate more efficiently and effectively

We have identified the main actors for the ECDC system, which are the Parent, ECD teacher, ECD Manager, Regional Coordinator and Provincial Liaison.

Problems and Thought Process

We opted for a web app as opposed to a desktop app as the majority of group members were familiar with web tech. For the framework/library, I had suggested Angular as I had been learning it for a while and because it was an MVC framework I felt it wouldn’t be too hard to pick up as they already were familiar with working with the .NET framework. The idea being that we should step out of our comfort zones and expand the tools and adapt to new challenges.

Unfortunately by the end, what was supposed to be a 5 man task was a 2 man product. Admittedly my project lead/team leader debut was not a smashing success but I learned a lot in the process. Such as adopting different managerial styles, multi-tasking, and time management to name a few. Coding wise, not being a native backend developer and wanting to not miss my milestones, I had to take insecure shortcuts such as passing ID information from the database using local storage.

Another problem was uploading images to my database to have more detailed profile cards for all users. Not being very good with PHP I was looking to use a firebase real-time database to keep images and use it to serve them when needed.

Lessons Learned/Moving Foward

As previously stated, I learned a lot of soft skills during the project. In this section, I will be focusing on hard skills. My takeaways were understanding how to pass data between components using ViewChild or Input decorator. Protecting paths using Auth guard to ensure no user has access to restricted pages and using services and directives to create more dynamic HTML markup.

I’m currently learning how to use MongoDB, The goal would be to implement it in this project and showcase the full project and not just the landing information site.
